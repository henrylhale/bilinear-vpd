# SPEC — Phase 1: Toy Tensor-Transformer with Synthetic Computational-Primitive DGP

## Project context

This is the first phase of a larger interpretability research project. The eventual goal is to apply adVersarial Parameter Decomposition (VPD; Bushnaq et al., Goodfire, May 2026) to a fully bilinear ("tensor") transformer trained on data generated by a simple program emulating primitives that transformers are known to model (bigram stats, induction, etc.), then construct a computational graph from the components of the decomposition.

This phase is just the data generator and the trained model. **Do not implement VPD or any decomposition machinery in this phase.** The goal is to have a working synthetic DGP and a small tensor-transformer that learns it well, so subsequent phases can apply VPD to a model whose ground-truth computational structure is known.

## Required reading

Before writing code, read or skim:

- The VPD paper: <https://www.goodfire.ai/research/interpreting-lm-parameters>. The relevant sections for this phase are Section 3.1 (target model architecture) and Appendix A.7 (training details). The bulk of the paper covers methods we'll apply later.
- The VPD code repository: <https://github.com/goodfire-ai/param-decomp>. Relevant for this phase: the model architecture and training scripts. Note that VPD's target model uses standard transformer architecture, not bilinear. We'll be modifying it.
- The Riggs/Dooms LessWrong post on tensor-transformer variants: <https://www.lesswrong.com/posts/hp9bvkiN3RzHgP9cq/tensor-transformer-variants-are-surprisingly-performant>. This contains the bilinear-attention form we'll use, and links to a code repo with bilinear-architecture implementations.
- The Pearce et al. paper "Bilinear MLPs enable weight-based mechanistic interpretability" (arXiv 2410.08417). For background on bilinear MLPs.

The bilinear-attention form is `(xQ₁K₁ᵀxᵀ) ⊙ (xQ₂K₂ᵀxᵀ) · v` — two QK pairs whose attention patterns are combined via elementwise product, then multiplied by V. **No softmax.** The MLP is bilinear in the standard Pearce et al. form: `(Wx) ⊙ (Vx)`, no elementwise nonlinearity.

## Phase goal

Implement two things:

1. A data generator for a synthetic language with three computational primitives: bigram statistics, skip-trigrams, and induction.
2. A **2-layer** fully-bilinear transformer (bilinear attention, bilinear MLP) trained on data from this generator, achieving good accuracy on the rule-governed predictions.

> **Layer-count note.** The original spec said 1 layer; corrected to 2 because induction requires composition across layers (a previous-token head in layer 1 feeding a copying head in layer 2).

"Good accuracy" specifically means: on the rule-governed token positions (positions where one of the three primitives determines the next-token distribution), the trained model's predicted distribution should be close to the true distribution. Concretely, target validation cross-entropy on rule-governed positions within ~5% of the entropy of the true distribution. The non-rule-governed positions (filler) will have higher loss because they're closer to uniform; this is fine.

## Data generator specification

### Vocabulary

Define typed vocabulary slots. Suggested initial sizes:

- `SUBJ`: 8 subject tokens (e.g., alice, bob, carol, dave, eve, frank, grace, henry).
- `VERB`: 12 verb tokens (e.g., sees, helps, finds, knows, likes, meets, calls, tells, asks, shows, gives, takes).
- `LOC`: 8 location tokens (e.g., beach, garden, lake, office, forest, kitchen, school, library).
- `ADJ`: 8 adjective tokens (e.g., big, old, new, small, red, blue, tall, short).
- `CONN`: 8 connective tokens (e.g., at, and, or, to, with, by, from, for).
- `FILLER`: 16 filler tokens (e.g., the, a, then, also, now, here, etc.).
- Special tokens: `<bos>`, `<eos>`, `<pad>`.
- Total vocabulary size: ~70 tokens. Make these configurable.

### Three computational primitives

Each primitive is a rule that fires at certain positions and determines the next-token distribution there.

**Primitive 1: bigram statistics (subject → verb).**

When the previous token is in `SUBJ`, the next token is sampled from a `SUBJ`-conditional distribution over `VERB`. Each subject has its own distribution over verbs, with most probability mass concentrated on a few verbs.

- For each subject `S`, sample once at DGP-construction time (with a fixed seed) a sparse distribution over `VERB`: pick 3 favored verbs and assign probabilities `(0.7, 0.2, 0.1)`. Store this table.
- When generating, if the previous token is `S ∈ SUBJ`, sample the next token from `S`'s verb distribution.

**Primitive 2: skip-trigrams ((LOC, ADJ) → CONN).**

When the previous token is an `ADJ` `A`, and there is a `LOC` `L` within the last `max_skip` (= 8) tokens, the next token is determined by the `(L, A)` pair according to a fixed rule table.

- Construct a rule table at DGP-construction time: pick `K = 4` distinct `(L, A)` pairs and assign each one a `CONN`. Store this as a dictionary `{(L_i, A_i): C_i}`.
- When generating: if the previous token is `ADJ A` and there's a `LOC L` within the last 8 tokens (looking back, picking the most recent `LOC` if multiple), and `(L, A)` is in the rule table, then the next token is the corresponding `CONN`.
- If multiple `LOC`s are within range and multiple `(L, A)` pairs match, prefer the most recent matching `L`.
- If no matching `(L, A)` pair exists, this primitive does not fire at this position.

**Primitive 3: induction.**

When the previous two tokens form a pair `(X, Y)` that has occurred earlier in the sequence (i.e., there was an earlier position where `X` was followed by `Y`), the next token is biased toward whatever followed the second occurrence of `X` further in the past.

Concretely: if the current position is `t` and the tokens at `t-1` and `t-2` are `Y` and `X` respectively (so the bigram ending at `t-1` is `X-Y`), look for the most recent earlier position `t' < t-1` where the same `X-Y` bigram appears. If such a `t'` exists, the token at `t` is biased toward the token at `t' + 1`.

> **Distribution choice (resolved during planning).** We use a *smoothed* delta: 0.9 mass on the induced token, 0.1 distributed uniformly over the rest of the vocabulary. This avoids pathological infinite-KL when the model is slightly off, while still strongly indicating the rule.

Implementation detail: for clarity, the induction rule fires only if there's a unique earlier occurrence of `X-Y`.

### Sequence generation

Each sequence is built left-to-right:

1. Start with `<bos>` and a few random fillers.
2. At each subsequent position, decide which primitive (if any) fires, with this precedence:
    - If the previous token is in `SUBJ`, fire bigram primitive (sample from `SUBJ`-conditional verb distribution).
    - Else if the previous token is in `ADJ` and a matching `LOC` is in range, fire skip-trigram primitive (deterministic next token).
    - Else if the previous-two tokens match an earlier bigram (uniquely), fire induction primitive (sample from the smoothed induced distribution).
    - Else, sample from a default unigram distribution (mostly fillers, with low probabilities for `SUBJ`/`VERB`/`LOC`/`ADJ` tokens to seed future primitive firings).
3. End with `<eos>` after some target sequence length (suggested 64–128 tokens).

The default unigram distribution should give roughly uniform low probability to "trigger" tokens (`SUBJ`, `ADJ`, `LOC`) so that primitives fire reasonably often, with the bulk of probability on `FILLER` tokens. Tune this so that across a training set, each primitive fires on roughly 5–15% of positions.

### Generator interface

The generator should be a Python class or set of functions that:

- Takes config (vocabulary sizes, rule tables, sequence length, etc.) and a seed.
- Provides a method to sample a batch of sequences as token-ID tensors.
- Provides a method that returns, alongside each sampled sequence, a per-position annotation indicating which primitive (if any) fired at each position. This is for evaluation.
- Stores all rule tables (subject→verb distributions, skip-trigram rules) as accessible attributes, so we can later query "is this circuit implementing rule X?" against ground truth.

Make all of this configurable but provide reasonable defaults that match the suggested vocab sizes.

## Model specification

### Architecture: 2-layer fully-bilinear transformer

A 2-layer decoder-only transformer with the following components per layer:

- Token embedding (`W_E`): standard embedding matrix (shared across layers; only one).
- Positional encoding: RoPE, applied to queries and keys.
- Bilinear attention (1 head per layer, no softmax):
  - Two Q matrices: `W_{Q1}, W_{Q2}`.
  - Two K matrices: `W_{K1}, W_{K2}`.
  - One V matrix: `W_V`.
  - One O matrix: `W_O`.
  - Compute two attention patterns: `A1 = (xW_{Q1} · R_τ · (xW_{K1})ᵀ)`, `A2 = (xW_{Q2} · R_τ · (xW_{K2})ᵀ)`, where `R_τ` is RoPE applied at position offset `τ`.
  - Combine: `A = A1 ⊙ A2`. Causal mask applied.
  - Output: `head_output = A · (xW_V · W_O)`. (No softmax anywhere.)
  - Implementation note: include `1/√d_head` scaling on each raw QK product for training stability.
- Residual connection after attention.
- Bilinear MLP:
  - Two matrices `W_M, W_N`.
  - Optional output projection `W_proj`.
  - Compute: `mlp_output = ((x · W_M) ⊙ (x · W_N)) · W_proj`.
- Residual connection after MLP.
- After the final layer: token unembedding (`W_U`) produces logits.

**No layernorm, no RMSNorm** — we want the architecture as cleanly bilinear as possible. (If training is unstable without normalization, add RMSNorm before each sublayer as a fallback, but try without first.)

### Suggested hyperparameters

- Residual stream dimension `d_model`: 64.
- Attention head dimension `d_head`: 32.
- MLP intermediate dimension: 128.
- Sequence length: 64.
- Vocabulary size: matches the DGP, ~70 tokens.

Tune these as needed for training stability and accuracy.

## Training

- Standard next-token-prediction objective (cross-entropy loss).
- AdamW optimizer.
- Learning rate: try `1e-3, 3e-4, 1e-4` and pick what works.
- Batch size: 64–256.
- Training duration: enough to converge — likely 10k–100k steps depending on data and hyperparameters.

## Evaluation

The training script should report, in addition to overall validation loss:

- Loss broken down by primitive type: bigram positions, skip-trigram positions, induction positions, filler positions.
- For each primitive, the KL divergence between the model's predicted distribution and the true conditional distribution on positions where that primitive fires.
- Top-1 accuracy on each primitive's positions.

The success criterion for this phase: on each of the three primitives, the model's predictions on rule-governed positions are close to the ground-truth distribution. Specifically, **KL divergence ≤ 0.1 nats on bigram and skip-trigram positions, and ≤ 0.3 nats on induction positions.**

## Implementation guidance

### Reuse existing code where possible

- The VPD repo has training infrastructure (model definitions, training loop, evaluation) for a standard transformer. Fork and modify, replacing the architecture with the bilinear version.
- The Riggs/Dooms repo (linked in the LessWrong post) has bilinear-architecture implementations. Use their bilinear MLP and bilinear attention modules if available; otherwise implement from the formulas above.
- Don't reinvent infrastructure — reuse model loaders, training loops, etc.

### Code organization

- A clean separation between the data generator (one module) and the model/training code (another module).
- The data generator should be importable and runnable standalone.
- The model code should be importable and runnable standalone.
- A top-level script that wires them together: instantiate generator, instantiate model, train, evaluate, save checkpoints.
- Save checkpoints frequently (every few thousand steps), since later phases will want to study the training dynamics.

### Testing

- Tests for the data generator: for each primitive, verify that on positions where the primitive should fire, the empirical next-token distribution matches the rule's distribution within a tolerance.
- Sanity check for the model: a forward pass on a small input should produce well-shaped logits without NaNs or infinities.
- Smoke test for training: a few hundred training steps on a small batch should reduce loss measurably.

## What to skip in this phase

Do not implement:

- VPD or any parameter decomposition.
- Any interpretability tools beyond the basic per-primitive evaluation.
- Multi-head models, models with more than 2 layers (we'll generalize later).
- Standard (non-bilinear) transformer baselines (we may add later for comparison).

## Deliverables

By the end of this phase, the repository should contain:

- A working data generator (with tests) that produces sequences with the three primitives and per-position annotations.
- A 2-layer fully-bilinear transformer implementation.
- A training script that trains the model on data from the generator.
- Evaluation showing the trained model meets the per-primitive accuracy targets.
- A short README explaining the setup, how to run training, and how to interpret the evaluation output.
- A saved checkpoint of a trained model, plus the data generator config used to train it, so subsequent phases can load both.

## Decisions made during planning (not in the original spec)

These are choices made during the planning conversation, recorded here so future phases can reference them:

- **2 layers** (was 1 in the original spec; corrected for induction).
- **Smoothed induction**: induction primitive's next-token distribution is `0.9 * δ_induced + 0.1 * uniform` rather than a hard delta.
- **Causal masking** in the bilinear attention (decoder-only LM).
- **Attention scaling**: each raw QK product is scaled by `1/√d_head` before the elementwise product.
- **No biases** on Q/K/V/O, MLP arms, or output projection — keeps the architecture cleanly bilinear.
- **Logging**: plain stdout + jsonl events; no WandB initially.
- **Codebase**: forked from `goodfire-ai/param-decomp` rather than written fresh, to make Phase 2 (applying VPD) easier.
- **RMSNorm fallback enabled.** Without normalization the induction circuit would not form (KL stuck at ~3.2 even after 50k steps at lr=1e-3). With pre-RMSNorm before each sublayer + a final pre-unembed norm, induction transitions through a phase change around step 7k–10k and converges to the target. The norms have no learnable affine — they preserve the parameter-bilinear structure.
